# Развертывание Ollama на Ubuntu (Без GPU)

Данная инструкция описывает процесс установки и запуска локальной нейросети через **Ollama** в консольном режиме на сервере Ubuntu, где доступно **60 ГБ оперативной памяти**, но отсутствует видеокарта (GPU).

Для проекта FLEX-N-ROLL PRO сервер Ollama будет работать как "Оракул" классификации лидов, и приложение будет иметь к нему доступ по адресу `http://host.docker.internal:11434/v1`.

---

## 1. Выбор модели для 60 ГБ RAM и работы на CPU

Поскольку у вас 60 ГБ оперативной памяти, технически вы можете загрузить очень большие модели. Однако при чисто процессорном (CPU) вычислении основным "узким местом" становится скорость генерации. Большие модели (32В+) на CPU работают очень медленно (1-2 токена в секунду).

**Идеальный баланс интеллекта и скорости для CPU:**
Мы рекомендуем **Qwen2.5 14B**.
- Модель достаточно "мозговита" для того, чтобы уверенно классифицировать разговоры на уровне GPT-4-mini.
- Она прекрасно понимает русский язык.
- Модель занимает около 10 ГБ ОЗУ, оставляя много свободной памяти.
- На хорошем многоядерном CPU она будет выдавать приемлемые 5–15 токенов в секунду, что оптимально для webhook-сервиса Битрикс24 (чтобы не отваливались по таймауту).

---

## 2. Установка Ollama (Скрипт одной строки)

Ollama — это индустриальный стандарт надежного локального запуска моделей, он идеально работает как фоновая `systemd` служба из коробки.

Подключитесь к вашему Ubuntu серверу и выполните команду:
```bash
curl -fsSL https://ollama.com/install.sh | sh
```
Скрипт автоматически скачает бинарники и настроит системную службу.

**Проверить, что Ollama запустилась успешно:**
```bash
sudo systemctl status ollama
```

---

## 3. Настройка сетевого доступа (Важно!)

По умолчанию Ollama принимает запросы только с `127.0.0.1`. Так как webhook-сервис FLEX-N-ROLL PRO запускается в Docker контейнере, ему нужен доступ к Ollama хост-машины через `host.docker.internal`.

Для этого нужно настроить переменную окружения `OLLAMA_HOST` в файле конфигурации systemd, чтобы она слушала `0.0.0.0` (все интерфейсы).

**1. Откройте override конфигурацию службы:**
```bash
sudo systemctl edit ollama.service
```

**2. Вставьте туда следующий блок кода (между закомментированными строками сверху и снизу):**
```ini
[Service]
Environment="OLLAMA_HOST=0.0.0.0:11434"
```

**3. Перезапустите службу:**
```bash
sudo systemctl daemon-reload
sudo systemctl restart ollama
```

---

## 4. Скачивание и запуск Qwen 2.5 14B

Выкачиваем рекомендованную модель. Этот процесс займет некоторое время в зависимости от скорости интернет-соединения сервера:
```bash
ollama pull qwen2.5:14b
```

Когда загрузка завершится, проверьте работу модели в интерактивном режиме:
```bash
ollama run qwen2.5:14b
```
*(Можете написать ей `Привет, как дела?`. Для выхода из оболочки чата нажмите `Ctrl+D`)*

### Другие модели на пробу:
- `ollama pull qwen2.5:32b` — умнее, но медленнее
- `ollama pull llama3.1` — быстрая Llama-3.1 8B (чуть похуже знает русский язык)

---

## 5. Проверка интеграции с проектом

Чтобы проект правильно соединялся с вашей локальной моделью Ollama, убедитесь, что в файле `.env` в корне проекта выставлены следующие параметры:

```dotenv
OPENAI_API_KEY=ollama
OPENAI_BASE_URL=http://host.docker.internal:11434/v1
OPENAI_MODEL=qwen2.5:14b
```

*Обратите внимание: Docker контейнер настроен с `extra_hosts: ["host.docker.internal:host-gateway"]`, что позволяет обращаться к `11434` порту хоста.*
