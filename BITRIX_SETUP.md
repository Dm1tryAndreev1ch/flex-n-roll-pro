# Полное руководство по настройке интеграции с Bitrix24 (с нуля)

Это пошаговое руководство позволит вам с нуля развернуть интеграцию FLEX-N-ROLL PRO с Bitrix24, получить токены авторизации и правильно привязать вебхуки и интерфейс аналитики.

---

## 1. Подготовка окружения (Файл `.env`)

1. Скопируйте файл `.env.example` в `.env`.
2. Зарегистрируйтесь на [ngrok.com](https://ngrok.com) и получите ваш `authtoken`.
3. Получите домен вашего портала Bitrix24 (например, `b24-abcde.bitrix24.by`).
4. Заполните в `.env` следующие поля (остальное для Битрикса пока оставляем пустым):
   ```env
   NODE_ENV=production
   BITRIX_PORTAL_DOMAIN=b24-abcde.bitrix24.by
   NGROK_AUTHTOKEN=ваш_длинный_ngrok_токен
   ```

## 2. Запуск контейнера (Старт ngrok)

Нам нужно запустить контейнер вебхука, чтобы он поднял ngrok-туннель, и мы узнали наш публичный адрес.

1. Запустите сервис вебхука:
   ```bash
   docker compose up -d --build webhook
   ```
2. Посмотрите логи, чтобы найти ваш `ngrok URL`:
   ```bash
   docker compose logs webhook --tail 20
   ```
   *Вы увидите строку вроде: `[server] ngrok tunnel: https://abc-123.ngrok-free.app`.*
   **Скопируйте этот URL** — он понадобится нам на следующем шаге.

---

## 3. Создание приложения в Bitrix24

1. Откройте ваш портал Bitrix24 под учетной записью администратора.
2. Перейдите в левом меню: **Разработчикам → Другое → Локальное приложение**.
3. Выберите тип: **Серверное**.
4. Заполните поля следующим образом (замените `ВАШ_NGROK_URL` на ссылку из шага 2):
   - **Путь вашего обработчика**: `ВАШ_NGROK_URL/install/callback` *(временно)*
   - **Путь для первоначальной установки**: `ВАШ_NGROK_URL/install/callback` *(временно)*
5. Установите права (Scope):
   - **CRM** — работа с лидами, сделками, контактами, дубликатами, таймлайном
   - **Задачи (tasks, tasks_extended)** — создание задач по лидам, SLA-мониторинг
   - **Чат и уведомления (im, imbot)** — автоответы клиентам, SLA-уведомления
   - **Телефония (telephony)** — обработка пропущенных звонков
   - **Пользователи (user)** — автоматическое получение списка сотрудников для маршрутизации
   - **Диск (disk)** — загрузка автосгенерированных КП (PDF)
6. Нажмите **Сохранить**.
7. Битрикс покажет два важных поля: **Код приложения** и **Ключ приложения**.

---

## 4. Сохранение ключей и обмен токенами

1. Откройте ваш файл `.env` и добавьте полученные ключи из Битрикса:
   ```env
   BITRIX_CLIENT_ID=ваш_код_приложения
   BITRIX_CLIENT_SECRET=ваш_ключ_приложения
   ```
   *(Убедитесь, что при копировании ключа браузер не включил автопереводчик — в ключе не должно быть кириллических букв!)*
2. Перезапустите вебхук, чтобы он подхватил новые ключи:
   ```bash
   docker compose up -d webhook
   ```
3. Откройте новую вкладку в браузере и перейдите по локальному адресу установки:
   👉 **`http://localhost:3000/install`**
4. Битрикс24 запросит подтверждение (окно авторизации). Нажмите кнопку разрешения.
5. Битрикс перенаправит вас обратно, и в браузере появится сообщение:
   `{"success": true, "message": "Приложение установлено! Токены сохранены в Redis."}`

> 🎉 **На этом этапе сервер получил постоянный доступ к вашему Bitrix24 API и автоматически установил системные вебхуки для CRM.**

---

## 5. Вывод дашборда аналитики в интерфейс Bitrix24

Теперь, когда системные вебхуки уже настроены (они работают по адресу `/webhook`), мы можем использовать интерфейс локального приложения внутри Битрикс24 для показа нашего React-дашборда `fnr-analytics`.

1. Вернитесь в Битрикс24 в настройки вашего локального приложения (`Разработчикам → Интеграции → Найти ваше приложение → Редактировать`).
2. Измените поле **Путь вашего обработчика** (самое первое поле) на чистый адрес:
   👉 `ВАШ_NGROK_URL/` *(без всяких окончаний)*
3. **Путь для первоначальной установки** можно оставить как есть (`.../install/callback`).
4. Нажмите **Сохранить**.

### Проверка результата:
Кликните на название вашего приложения в левом меню Битрикс24. Внутри портала мгновенно откроется вкладка React-аналитики!

---

## 6. Проверка статуса

Если вы сомневаетесь, всё ли работает, откройте в браузере:
👉 **`http://localhost:3000/install/status`**

Идеальный ответ:
```json
{
  "oauth": {
    "installed": true,
    "domain": "ваш-домен.bitrix24.by",
    "hasRefresh": true
  },
  "ngrok": "https://abc-123.ngrok-free.app",
  "ready": true
}
```

## 7. Мониторинг (Grafana + Prometheus)

После того как основа запущена, вы можете запустить систему аналитики метрик:
```bash
docker compose -f docker-compose.monitoring.yml up -d
```
Grafana будет доступна на вашем хосте по адресу `http://localhost:3001` (пароли по умолчанию `admin / admin`, если вы не меняли их в `.env`).
