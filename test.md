# Сценарий сквозного тестирования: Обработка лида из Bitrix24

Данный документ описывает пошаговый процесс ручного тестирования конвейера обработки лидов: от создания сущности в CRM до работы AI, подбора менеджера и выставления задачи.

## 1. Инициация: Создание лида в Bitrix24

Тестирование начинается в интерфейсе Bitrix24. Лид можно создать вручную или сымитировать входящее сообщение через Открытые Линии (чат-боты / соцсети). 

### Как создать лид вручную
1. Перейдите в раздел **CRM** → **Лиды** в вашем Bitrix24.
2. Нажмите кнопку **«Создать»** или **«Добавить»**.
3. Заполните тестовые данные. Чтобы увидеть максимум возможностей AI, напишите неструктурированный текст в поле комментария или описания.
   - **Имя / Фамилия:** Тест Тестов
   - **Телефон / Email:** +79001234567 / test@flexnroll.com
   - **Название лида:** Запрос из тестового сценария
   - **Комментарий (или Дополнительно об источнике):**  
     *Пример текста для классификации:* «Здравствуйте, нам нужен тираж 10 000 самоклеящихся этикеток для бутылок вина (PET). Сроки горят, нужно сделать до пятницы! Бюджет около 50 тысяч рублей.»
4. Сохраните лид.

---

## 2. Передача данных: Что куда передается?

Как только вы нажмете «Сохранить», Bitrix24 генерирует событие добавления нового лида.

1. **Исходное событие:** Срабатывает исходящий вебхук Bitrix24 на событие `ONCRMLEADADD`.
2. **Отправка Webhook:** Bitrix24 по REST делает `POST`-запрос на URL вашего сервиса (например, `https://ваша-домен/webhook`).
3. **Payload (тело запроса):** Передаются `id` созданного лида и название эвента. 
4. **Дозагрузка (Enrichment):** Сервис Node.js (файл `webhook.js`) принимает запрос. Так как вебхук отдает только ID, сервис делает обратный вызов (через метод `getLead`), чтобы вытянуть полное описание, комментарии и контактные данные.

---

## 3. Обработка данных сервисом: Что как делается?

Движок приложения (Node.js) берет управление на себя и прогоняет данные по внутреннему пайплайну:

### Шаг 3.1: AI-классификация (LMStudio)
Сервис вызывает локальный AI-движок (`services/lmstudio.js`), передавая в промпте текст лида.
**Нейросеть извлекает и типизирует данные:**
- **Интент (намерение)**: Например, `quote_request` (Запрос КП).
- **Продукт**: Например, `self_adhesive_pet`.
- **Срочность**: Высокая (из-за «сроки горят»).
- **Приоритет (SLA)**: Рассчитывается от 1 (HOT) до 3 (Обычный).
- **Извлеченные сущности**: Из текста достаются тираж (10000), бюджет (50000) и требования к материалу.

### Шаг 3.2: Маршрутизация (Routing)
Модуль `routing.js` решает, на кого из сотрудников назначить этого лида.
- На основе интента (`quote_request`) выбирается пул операторов, в данном случае — `sales` (Продажи).
- Если бы клиент жаловался на брак, пул был бы `quality` (Рекламация).
- Происходит **Round-Robin** балансировка: система берет счетчик из Redis (или локального JSON файла) и берет "следующего" по очереди менеджера для равномерного распределения нагрузки.

---

## 4. Результат: Что меняется в Bitrix24

Когда сервис отработал классификацию и выбрал ответственного, он начинает вносить изменения обратно в Bitrix24:

### Шаг 4.1: Обогащение полей лида
Сервис (`services/bitrix.js > updateLead`) прописывает извлеченные классификатором данные в карточку лида:
- Задается "Ответственный менеджер" (тот, кого подобрал Round-Robin).
- Заполняются пользовательские поля (UF-поля): Тип запроса, Продукт, Срочность и AI-приоритет.
- В поле `OPPORTUNITY` (Сумма/Ожидаемая выручка) автоматически проставляются «50000» с валютой RUB, если AI смог достать бюджет из текста.
- Добавляется внутренний комментарий с пояснением того, как AI классифицировл обращение.

### Шаг 4.2: Постановка задачи с жестким SLA
Сервис (`services/bitrix.js > createTask`) создает задачу, привязанную к этому лиду:
- **Название:** `[Лид #ID] [HOT] Запрос КП — Самоклейка (PET)`.
- **Описание задачи:** В красивом виде структурируется вся табличка: интент, срочность, телефон, бюджет и выжимка текста.
- **Ответственный:** Тот же менеджер из пункта 3.2.
- **Дедлайн (Крайний срок):** Высчитывается автоматически в зависимости от приоритета (например, для горячих лидов — ответ в течение 1–4 часов от текущего момента).

### Шаг 4.3: Автоответ (только для чатов / Open Channels)
Если лид пришел из чата (`ONIMCONNECTORMESSAGEADD`), сервис также дернет метод отправки сообщения в Открытую Линию — например, «Ваша заявка на расчет 10.000 этикеток принята, менеджер такой-то ответит через 20 минут».
